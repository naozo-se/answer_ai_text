設計書やチーム内での共有資料に記載しやすい、キーワードレベルの短い箇条書きですね。

システム構成の懸念点としてインフラエンジニアやバックエンドエンジニアに見せれば、「確かにそれは大変だ（だからWebSocketにしよう）」と一発で伝わるキーワードをいくつかピックアップしました。

用途に合わせて使い分けてみてください。

### 【インフラ・ネットワークの懸念点】
* **動的UDPポートの大量消費とFW（ファイアウォール）管理**
* **STUN/TURNサーバーの追加構築と運用コスト**
* **ステートフル通信における負荷分散（ロードバランサ）の複雑化**
* **企業内プロキシによるUDP通信の遮断リスク**

### 【実装・アプリケーションの懸念点】
* **B2BUA（Back-to-Back User Agent：音声中継）実装の高難易度化**
* **SDP再ネゴシエーション（再接続時）の複雑な状態管理**
* **サーバーサイドWebRTCライブラリの依存関係と保守性**
* **パケットロス時のアプリケーション側でのエラーハンドリング**
